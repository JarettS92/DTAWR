Overview
Gives step by step instructions on how to use these tools and reports

WARNING: Only for use by Dynatrace employees. Do not distribute to customers.

This section is to tell the user how to go about using the Reports functions

1. Make sure you configure your Tenants first! (top right corner)

2. Navigate to the report you want to run

3. Fill in any required data (dates, tags)

4. Run the report **Some may download a .csv file on completion**

Tag Duplicator
Copies a tag and sends it to the same or a different environment with the same or a different name

WARNING: Only for use by Dynatrace employees. Do not distribute to customers.

REQUIRED TOKEN PERMISSION:

Specify an environment and tag, this will be the tag that that is duplicated

Specify a second environment and tag, this will be where the duplicate tag is sent.

You can use the same tag name as before if a tag with that name does not exist in the second environment.

You can use the same environment for the second as the first, but the new tag name should be different.

Business Hour KPIs

Specify the timeseries metric id you would like to use. Available SaaS and Managed timeseries metrics.

Custom and plugin metrics should work, but the timeseries ids will not be in the above links.

DON'T use the metric display name, multiple metrics in Dynatrace use the same name.

Starter Tags
Adds valuable tags to Dynatrace

WARNING: Only for use by Dynatrace employees. Do not distribute to customers.

Specify an environment and which tagging rules you'd like to start off with.

Each rule will create a contextual tag that tags the entity type with its own name, with the exception of the IP Address rule, which tags a host with its IP

Example: Service rule tags an entity with Service: [Entity's Detected Service Name]

Why would you want to do this? Using the tags this program creates allows you to run API requests against a single entity when you pass the new tags with the entity's name in the tag parameter of the request!

Note: If you intend to run ARC, you need to run the Host autotag rule before hand.


Check which autotags you would like to apply.
